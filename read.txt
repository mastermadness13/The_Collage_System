# Full Fix Plan - COMPLETE ✅

## Problems Found
1. **Missing `db.py`** - App imports `db` but doesn't exist
2. **Incomplete timetable route** - No API endpoints (only static template)
3. **Frontend hardcoded data** - No backend integration
4. **DB schema mismatch** - courses/teachers use `department TEXT`, not `department_id`

## Fixes Applied

### ✅ 1. Created `db.py`
- Database connection manager via Flask `g` context
- `get_db()` - returns cached connection with `sqlite3.Row` factory
- `close_db()` - teardown handler
- `init_app(app)` - registers teardown

### ✅ 2. Updated `app.py`
- Added `import db` + `db.init_app(app)` after app creation

### ✅ 3. Rewrote `routes/timetable.py`
- Added `url_prefix='/timetable'` to Blueprint (fixes root `/` collision)
- **11 REST API endpoints:**

| Endpoint | Method | Auth | Purpose |
|----------|--------|------|---------|
| `/timetable/` | GET | login_required + admin_department_required | Render page |
| `/timetable/api/entries` | GET | login_required + admin_department_required | List entries with filters (semester, department_id, section) |
| `/timetable/api/entries` | POST | login_required + courses_timetable_admin_required | Create entry with conflict check |
| `/timetable/api/entries/<id>` | PUT | login_required + courses_timetable_admin_required | Update entry |
| `/timetable/api/entries/<id>` | DELETE | login_required + courses_timetable_admin_required | Delete entry |
| `/timetable/api/check-conflict` | POST | login_required | Check conflict without saving |
| `/timetable/api/courses` | GET | login_required | Dropdown (filters by department name via subquery) |
| `/timetable/api/teachers` | GET | login_required | Dropdown (filters by department name via subquery) |
| `/timetable/api/rooms` | GET | login_required | Dropdown (filters by department_id) |
| `/timetable/api/departments` | GET | super_admin_required | Department list |
| `/timetable/api/periods` | GET | login_required | Time slots from period_settings |

### ✅ 4. Updated Frontend `templates/timetable/timetable.html`
- Removed all hardcoded JS arrays (`courses`, `teachers`, `rooms`, `entries`)
- Added `loadReferenceData()` - fetches courses/teachers/rooms from API
- Added `loadEntries()` - fetches timetable entries from API
- `renderTimetable()` - now async, fetches data on each render
- `saveEntry()` - POST/PUT to API with server-side conflict check
- `executeDelete()` - DELETE via API
- Department filter triggers re-render
- Preserved all UI: modal, semester pills, searchable dropdowns, color coding, toast, theme toggle, global search

### ✅ 5. Fixed API Schema Mismatches
- courses: `department TEXT` → subquery `(SELECT name FROM departments WHERE id = ?)`
- teachers: `department TEXT` → subquery `(SELECT name FROM departments WHERE id = ?)`
- rooms: `department_id` → works directly
- period_settings: uses `label` column (not `name`) + `sort_order` (not `code`)

### ✅ 6. CSS Already Exists
- `static/css/timetable.css` - 1147 lines, comprehensive styling with dark/colorful themes
- Referenced correctly in HTML as `/static/css/timetable.css`

### ✅ 7. Database Connection Verified
- Config uses `clean_data.db` (147KB, populated with departments, rooms, users, courses, teachers)
- `db.get_db()` works correctly with app context
- 7 departments, rooms with data, period_settings with A/B/C periods
